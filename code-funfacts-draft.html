Fun Facts App > Create XML Layout



1. Create new project

Application name is the name of your app that appears on Google Play



2. Specify form factor

Targeting older SDK/API versions can cater to more devices, but development would come with fewer features



3. Select template for first activity

Let's start with a blank activity



4. Identify your new activity

Activity name is the Java class name (the naming convention is _______Activity). Layout name is the XML file name. Title is the name displayed on the label of the user's screen.



5. Wait for project's build to complete

After the project is built successfully, you will see the screen below
   - Project directory: to navigate between files in your project
   - Code editor: to edit your code
   - XML display preview: to view changes to the activity's XML file instantly
   - Debugger: to identify and debug code errors
   - "Run" button: to build and run the project as an app
   - Highlighted files (FunFactsActivity and activity_fun_facts.xml):
      Java and XML files for this activity




6a. First test run by connecting Android device (method 1)

An empty shell is provided to the app at initial project creation, with just a "Hello world!" textView. Let's try running this app with an Android device (phone or tablet) connected to our computer via USB cable

Click on the "Run" button > Select connected device > "OK"

If your device is not detected, troubleshoot here
(On your device: open Settings > Developer Options > enable USB debugging)


6b. First test run by launching emulator (method 2)

An empty shell is provided to the app at initial project creation, with just a "Hello world!" textView. Let's try running this app with an Android Virtual Device (AVD) on our computer, i.e. an emulator

Click on the "Run" button > Click on "..." button as below to create a new emulator


Click on "Create.." > Fill in the specifications as below (tick "Use Host GPU")


Select the new emulator from the drop-down list > "OK"
